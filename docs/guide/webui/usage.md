# WebUI 完整使用指南

本指南逐一讲解 WebUI 各功能模块，帮助你充分利用 biliup 的管理能力。

## 访问 WebUI

启动 biliup 后，根据部署方式访问对应地址：

| 部署方式 | 默认访问地址 |
|---|---|
| 本地直接运行 | `http://localhost:19159` |
| Docker | `http://<服务器IP>:19159` |
| 自定义端口 | `http://<地址>:<端口>` |

首次访问会跳转到登录页，使用启动参数 `--auth 用户名:密码` 设置的凭据登录。

> ⚠️ 如未设置 `--auth`，部分功能可能受限或完全开放，请勿在生产环境不设认证直接暴露端口。

---

## 页面总览

登录后进入的主界面包含以下模块（左侧导航栏）：

```
┌─────────────────────────────────┐
│  🏠 仪表盘 (dashboard)        │
│  📡 录播管理 (streamers)      │
│  📤 投稿管理 (upload-manager)  │
│  📋 任务管理 (job)            │
│  📜 历史记录 (history)         │
│  📊 运行状态 (status)          │
│  📝 日志查看 (logviewer)       │
│  ⚙️  空间配置                 │
└─────────────────────────────────┘
```

## 仪表盘 (Dashboard)

**路径：** `/dashboard`

全局概览页，一目了然地展示当前系统状态。

### 主要信息卡片

| 卡片 | 内容 |
|---|---|
| 活跃录制数 | 当前正在录制的直播间数量 |
| 等待上传数 | 已录制完成待上传的视频数 |
| 今日录制时长 | 当天累计录制时间 |
| 账号状态 | B站登录账号是否有效、是否过期 |

### 快捷操作

- **添加主播** — 快速跳转到录播管理的新增页面
- **查看日志** — 快速跳转到日志查看器
- **刷新状态** — 手动触发状态刷新（自动轮询间隔可在空间配置中调整）

> 💡 仪表盘数据默认每 30 秒自动刷新，可在「空间配置 → 全局设置」中调整轮询间隔。

## 录播管理 (Streamers)

**路径：** `/streamers`

核心功能模块，管理所有需要录制的直播间。

### 添加直播源

点击「新增」按钮，支持以下方式的直播源添加：

**方式一：直接粘贴房间链接（推荐）**

支持各平台的完整 URL，biliup 会自动识别平台：

```
# B站
https://live.bilibili.com/123456

# 斗鱼
https://www.douyu.com/123456

# 虎牙
https://www.huya.com/123456

# Twitch
https://www.twitch.tv/username

# YouTube
https://www.youtube.com/watch?v=xxxxx
https://youtu.be/xxxxx
```

**方式二：手动选择平台 + 填写房间号**

适用于链接识别异常的情况。

### 主播卡片说明

每个已添加的主播以卡片形式展示：

```
┌─────────────────────────────────────────┐
│  🔴 直播中  │  主播名 (平台)           │
│─────────────────────────────────────────│
│  房间标题：xxxxxxxxxxxxxxxxxxxx         │
│  开播时间：2025-03-25 12:46:42      │
│  录制状态：录制中 (已录 1.2 GB)       │
│  ┌────────┐  ┌────────┐  ┌────────┐│
│  │  编辑   │  │  删除   │  │  日志   ││
│  └────────┘  └────────┘  └────────┘│
└─────────────────────────────────────────┘
```

### 单个主播配置项

点击主播卡片的「编辑」按钮，可配置该主播的独立参数（优先级高于全局配置）：

| 配置项 | 说明 |
|---|---|
| 录制开关 | 临时禁用/启用该主播的录制 |
| 上传插件 | 覆盖全局设置，单独指定上传方式（`Noop` = 不自动上传 / `biliup-rs` = 自动上传） |
| 投稿模板 | 指定使用的投稿配置模板 |
| 文件名模板 | 覆盖全局的 `filename_prefix` |
| 分段大小/时长 | 覆盖全局的视频分段规则 |
| 弹幕录制 | 是否录制该主播的弹幕（需安装 ffmpeg） |
| Cookie/Token | 部分平台需要单独设置认证信息 |

### 批量操作

- **全选** — 批量启用/禁用录制
- **批量应用模板** — 为多个主播统一指定投稿模板
- **批量删除** — 移除多个主播（不会删除已录制的文件）

## 投稿管理 (Upload Manager)

**路径：** `/upload-manager`

管理 B站账号和投稿模板，是自动上传和投稿的配置中心。

### 账号管理

#### 添加 B站账号

1. 进入「投稿管理」→ 点击左上角「新增」
2. 选择「扫码登录」
3. 使用哔哩哔哩 APP 扫描屏幕上显示的二维码
4. 手机确认登录后，页面自动刷新，显示账号信息

> ✅ 登录成功后，Cookie 会自动保存，无需重复登录。Cookie 有效期通常为 1-3 个月。

#### 多账号管理

biliup 支持添加多个 B站账号：
- 不同主播可以指定不同的投稿账号
- 账号列表中显示每个账号的 UID、昵称、Cookie 有效期
- 点击「刷新登录」可重新扫码续期

### 投稿模板

投稿模板是 biliup 的核心概念：**将投稿参数打包成一个可复用的配置**。

#### 创建模板

点击「模板管理」→「新建模板」，填写以下参数：

**基础信息：**

| 参数 | 说明 | 示例 |
|---|---|---|
| 模板名称 | 模板的标识符，便于区分 | `默认投稿`、`游戏区投稿` |
| 投稿账号 | 使用哪个 B站账号投稿 | （选择已登录的账号）|
| 分区 | 视频投稿的分区 | `游戏 → 电子竞技` |
| 标题格式 | 支持变量替换 | `{title} - {date}` |
| 简介 | 视频简介，支持变量 | `录制自 {streamer} 的直播间` |
| 标签 | 视频标签 | `直播录制, {streamer}` |
| 封面 | 封面获取方式 | 自动使用直播封面 / 手动指定 |

**高级参数：**

| 参数 | 说明 |
|---|---|
| 动态 | 投稿时同步发布的动态内容 |
| 延迟发布 | 设定投稿后多久才公开（小时） |
| 合集 (Season) | 将视频加入指定合集 |
| 互动视频 | 启用互动视频功能 |
| 杜比音效 | 开启杜比音效（需 biliup-rs 上传插件）|
| 充电专属 | 设为充电专属视频 |

**支持的变量：**

| 变量 | 替换为 |
|---|---|
| `{streamer}` | 主播名 |
| `{title}` | 直播标题 |
| `{date}` | 录制日期 `YYYY-MM-DD` |
| `{time}` | 录制时间 `HH:MM:SS` |
| `{url}` | 直播间链接 |

#### 模板与主播的绑定

在「录播管理」中编辑主播 → 选择「投稿模板」→ 指定该主播使用的模板。

绑定后，该主播录制完成并启用自动上传时，会使用对应模板的参数进行投稿。

## 任务管理 (Jobs)

**路径：** `/job`

查看和管理当前正在执行或排队中的任务。

### 任务类型

| 任务类型 | 说明 |
|---|---|
| 录制任务 | 正在录制直播间的任务 |
| 上传任务 | 正在上传视频到 B站的任务 |
| 后处理任务 | 分段后处理（如弹幕压制、格式转换）正在执行 |
| 投稿任务 | 上传完成后自动投稿的任务 |

### 任务操作

- **查看进度** — 点击任务可查看详细进度（上传百分比、当前速率等）
- **取消任务** — 中止正在进行的任务（录制任务取消后不会生成文件）
- **重试任务** — 上传失败的任务可手动触发重试
- **查看日志** — 快速跳转到该任务的日志记录

> ⚠️ 录制任务一旦取消，已录制的临时文件可能会被清理，请谨慎操作。

## 历史记录 (History)

**路径：** `/history`

查看所有已处理完成的录制和上传记录。

### 记录列表字段

| 字段 | 说明 |
|---|---|
| 主播名 | 来源主播 |
| 标题 | 录制时获取的直播标题 |
| 文件大小 | 录制文件的尺寸 |
| 录制时间 | 开播 → 下播的时间范围 |
| 上传状态 | 未上传 / 上传中 / 已上传 / 投稿完成 |
| 操作 | 查看详情 / 手动触发上传 / 删除记录 |

### 过滤与搜索

- **按主播过滤** — 只看某个主播的历史记录
- **按状态过滤** — 只看未上传 / 已上传的记录
- **按时间范围过滤** — 查看指定日期范围内的记录
- **关键词搜索** — 按标题关键词搜索

### 手动触发上传

对于未启用自动上传、或自动上传失败的主播，可以在历史记录中：
1. 找到对应的记录
2. 点击「上传」按钮
3. 选择投稿模板
4. 确认后开始上传

## 运行状态 (Status)

**路径：** `/status`

实时展示 biliup 后台服务的运行状态。

### 展示信息

| 信息类别 | 内容 |
|---|---|
| 系统资源 | CPU 使用率、内存占用、磁盘剩余空间 |
| 录制线程 | 当前活跃的录制线程数和配置的上限 |
| 上传线程 | 当前活跃的上传线程数和配置的上限 |
| 网络速率 | 当前下载速率和上传速率 |
| 版本信息 | 当前运行的 biliup 版本号 |

### 服务控制

- **重启服务** — 软重启 biliup（不中断系统服务）
- **停止服务** — 完全停止 biliup（需手动重新启动）

> ⚠️ 停止服务后 WebUI 将不可访问，需通过命令行或系统服务管理器重新启动。

## 日志查看 (Log Viewer)

**路径：** `/logviewer`

在浏览器中直接查看 biliup 的运行日志，无需 SSH 到服务器。

### 日志级别

| 级别 | 说明 |
|---|---|
| ERROR | 错误，需要关注 |
| WARN | 警告，可能影响功能但不致命 |
| INFO | 一般信息，记录正常操作流程 |
| DEBUG | 调试信息，排查问题时开启 |

### 使用技巧

- **级别过滤** — 选择只显示某一级别及以上的日志
- **关键词搜索** — 在日志中搜索特定关键词（如房间号、错误码）
- **自动滚动** — 开启后新日志会自动滚动到可见区域
- **下载日志** — 将当前日志下载到本地，用于反馈问题

> 💡 提交 GitHub Issue 时，请附上相关时间段的日志文件，能大幅提高问题解决速度。

## 空间配置

点击左下角「空间配置」进入配置中心，包含三个子页面：

### 全局设置

影响所有主播的默认行为，详见[全局设置文档](/guide/introduce/Config/GlobalConfig.html)。

关键参数速查：

| 参数 | 默认值 | 说明 |
|---|---|---|
| 下载器 | `stream-gears` | 可选 `streamlink` / `ffmpeg` / `sync-downloader` |
| 分段大小 | - | 按文件大小分段（Byte） |
| 分段时长 | - | 按录制时长分段（`HH:MM:SS`） |
| 文件名模板 | `{streamer}%Y-%m-%dT%H_%M_%S` | 录制文件命名格式 |
| 下载线程池大小 | `5` | 同时录制的最大直播间数 |
| 下播延迟检测 | `0` | 延迟 N 秒再次确认下播 |
| 直播事件检测间隔 | - | 单个主播的检测间隔（秒） |

### 各平台设置

针对特定平台的配置，详见[各平台设置文档](/guide/introduce/Config/liveconfig.html)。

常见需要单独配置的平台：
- **哔哩哔哩** — 直播流协议（flv / hls_fmp4）、Cookie 文件、API 反代地址
- **斗鱼** — CDN 优选
- **虎牙** — CDN 优选
- **YouTube** —  cookies 文件（用于访问受限内容）

### 开发者选项

面向进阶用户，详见[开发者选项文档](/guide/introduce/Config/developerOptions.html)。

主要功能：
- **钩子系统 (Hooks)** — 在录制前 / 分段后 / 上传前 / 上传后执行自定义脚本
- **Webhook** — 上传完成后向指定 URL 发送 POST 通知
- **分段处理器** — 配置 `remux:mp4` 或自定义 Python 脚本
- **日志级别** — 调整日志详细程度
- **自定义 WebUI 目录** — 使用自行修改的 WebUI 前端

## 常见问题

### 无法访问 WebUI

1. 确认 biliup 已正常启动（查看终端输出）
2. 确认访问的端口正确（默认 19159，可通过 `--port` 修改）
3. 如部署在远程服务器，确认防火墙已放行对应端口
4. 如在 Docker 中运行，确认已正确映射端口（`-p 19159:19159`）

### 修改配置后不生效

全局配置和平台配置的修改**必须重启 biliup** 才能生效。在「运行状态」页面点击「重启服务」，或在终端中重启。

### 忘记 WebUI 密码

在启动命令中重新指定 `--auth` 参数，或查看启动时的终端输出（部分部署方式会在首次启动时显示随机生成的密码）。

### WebUI 响应缓慢

- 检查服务器资源（CPU / 内存 / 磁盘）是否充足
- 降低日志级别（DEBUG 级别会产生大量日志影响性能）
- 减少同时录制的直播间数量
