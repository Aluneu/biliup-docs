# 帮助与反馈

## 提问的艺术

> [!IMPORTANT]
> 当你提出一个技术问题时，最终是否能得到有用的回答，往往取决于你所提问和追问的方式。

> [!TIP]
> **好的提问应包含：**
> - 你的环境信息（操作系统、biliup 版本号、安装方式）
> - 你做了什么操作（具体命令或 WebUI 步骤）
> - 期望的结果是什么
> - 实际发生了什么（附带错误日志！）
> - 你已经尝试过哪些解决方案

> [!CAUTION]
> 请尊重开发者的无偿付出。我们欢迎友好且有建设性的交流。

---

## 反馈渠道

### GitHub Issues（推荐）

在 [biliup/biliup Issues](https://github.com/biliup/biliup/issues) 提交 Issue 是最高效的反馈方式。

提交前请：
1. 搜索已有 Issue，确认是否为已知问题
2. 使用 Issue 模板
3. 附上相关日志（`ds_update.log`、`upload.log` 等）

### 即时交流

- QQ 群 / Telegram 群（详见 [biliup 社区](https://biliup.me)）
- 在群内提问前，建议先阅读本文档
