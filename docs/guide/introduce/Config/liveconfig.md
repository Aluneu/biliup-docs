# 各平台设置

此页面记录各直播平台的独立配置项。以下配置均在 WebUI 的「空间配置」→「各平台设置」中修改。

---

## 哔哩哔哩

| 配置项 | 说明 | 推荐值 |
|--------|------|--------|
| Cookie 文件 (`bili_cookie_file`) | 选择已登录的 B站账号 | 必填 |
| 直播流协议 (`bili_protocol`) | HLS 流格式 | `hls_fmp4`（国内）/ `hls` |
| 直播主 API (`bili_liveapi`) | 境外服务器录制时的反代地址 | 留空或填入反代地址 |
| 录制弹幕 | 是否录制直播弹幕 | 按需开启 |

### 国内机器

- 下载器使用 `streamlink`
- 需要安装 FFmpeg
- Cookie 文件必选
- 协议选 `hls_fmp4`

### 境外机器

- 需要自备国内反代服务器
- 反代目标：`https://api.live.bilibili.com`
- 反代成功后打开该地址应显示 B站 404 页面
- 将反代地址填入「直播主 API」
- 可使用宝塔、Nginx、Caddy 等工具搭建反代

---

## 虎牙 / 斗鱼

| 配置项 | 说明 |
|--------|------|
| CDN 选项 | 如果视频频繁断流，尝试切换 CDN 线路 |

可使用默认下载器 `stream-gears`，不需要额外安装 FFmpeg。

---

## 抖音

需配置 Cookie 才能使用主页链接或避免风控。

---

## 快手

- 需中国大陆 IPv4 家宽 IP
- 24 小时内单直播间最多 120 次检测请求
- 风控较为严格，暂无完美解决方案

---

## Twitch

- 可配置登录信息以获取更高画质
- 推荐录制回放（VOD）而非直播流
- 支持弹幕录制

---

## YouTube

- 可配置登录信息
- 推荐录制回放（VOD）
- 可配置回放下载日期范围

---

## 其他平台

大部分平台使用默认 `stream-gears` 下载器即可，无需额外配置。如有特殊需求，请在 **biliup 社区** 或 [GitHub Issues](https://github.com/biliup/biliup/issues) 中反馈。
